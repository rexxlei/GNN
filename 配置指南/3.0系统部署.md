# 系统部署与配置完整指南

本文档整合了 3.0 系统部署流程及 Ubuntu 工控机网络配置修复方案。

---

## 1. 文件位置说明

- **配置文件**：`config` 文件夹应放置在 `/home/cotek` 目录下。
- **驱动程序**：其他驱动程序请参考 `drivers` 文件夹。

---

## 2. 启动前准备工作

### 2.1 更换系统源

1. **查看当前系统源**：
   ```bash
   sudo cat /etc/apt/sources.list
   ```

2. **更换为国内源 (清华/科大源)**：
   运行以下命令将源替换为中科大镜像源：
   ```bash
   sudo sh -c 'cat > /etc/apt/sources.list << EOF
   deb https://mirrors.ustc.edu.cn/ubuntu/ focal main restricted universe multiverse
   deb https://mirrors.ustc.edu.cn/ubuntu/ focal-updates main restricted universe multiverse
   deb https://mirrors.ustc.edu.cn/ubuntu/ focal-backports main restricted universe multiverse
   deb https://mirrors.ustc.edu.cn/ubuntu/ focal-security main restricted universe multiverse
   EOF'
   ```

### 2.2 运行环境配置脚本

该脚本用于开启 `vcan0`、配置 WiFi 网口及雷达网口。

1. **赋予脚本执行权限**：
   ```bash
   # 请在 setup_tracer.sh 所在目录运行
   sudo chmod +x setup_tracer.sh
   ```

2. **执行脚本**：
   ```bash
   sudo ./setup_tracer.sh
   ```

### 2.3 编译并安装

1. **赋予 `rosout` 权限**：
   ```bash
   sudo chmod +x /opt/ros/noetic/lib/rosout/rosout
   ```

2. **添加环境变量**：
   在 `~/.bashrc` 文件末尾添加以下内容：
   ```bash
   source ~/cotek/cotek_ws/devel/setup.bash
   ```

3. **编译并安装到系统**：
   ```bash
   cd ~/cotek/cotek_ws/build
   sudo make
   sudo make install
   ```

## 3. 工控机网络配置修复指南

**目标**：删除硬编码的静态 IP，并恢复网络设置的图形化界面（GUI）管理权限。

### 3.1 问题背景

当前工控机的网络配置文件 `/etc/netplan/50-cotek-init.yaml` 可能存在以下问题：
1. **渲染器 (Renderer) 设置问题**：被设置为 `networkd`，导致 GUI 无法管理网络。
2. **静态 IP 硬编码**：`192.168.93.11` 被写死。

### 3.2 修复步骤

#### 第一步：备份配置文件 (强烈建议)
```bash
cd /etc/netplan/
sudo cp 50-cotek-init.yaml 50-cotek-init.yaml.bak
```

#### 第二步：编辑配置文件
使用 `nano` 编辑器打开：
```bash
sudo nano 50-cotek-init.yaml
```

**修改要点**：
1. 将 `renderer` 从 `networkd` 改为 `NetworkManager` (注意大小写)。
2. 针对 `eth3` 网口：启用 DHCP，并注释掉写死的静态 IP。

**修改示例**：

*修改前*：
```yaml
renderer: networkd
# eth3 配置略... (包含静态 IP: 192.168.93.11/24)
```

*修改后*：
```yaml
renderer: NetworkManager
eth3:
  dhcp4: true
  # addresses: [192.168.93.11/24]  <-- 注释掉静态 IP
```

> **Nano 编辑器操作提示**：
> - `Ctrl + O`：保存文件
> - `Enter`：确认文件名
> - `Ctrl + X`：退出编辑器

#### 第三步：应用更改
```bash
sudo netplan apply
```

### 3.3 结果验证

查看 `eth3` 的 IP 地址：
```bash
ip addr show eth3
```

**成功标准**：
- 不再出现 `192.168.93.11`。
- 只保留路由器分配的动态 IP (如 `192.168.110.xxx`)。

**如果旧 IP 仍然存在**：
- 手动删除：
  ```bash
  sudo ip addr del 192.168.93.11/24 dev eth3
  ```
- 或重启系统：
  ```bash
  sudo reboot
  ```

## 4. VPN 配置

### 4.1 安装 Clash

1. **解压安装**：
   解压 U 盘中的 Clash 压缩包，并在终端中运行：
   ```bash
   sudo mv Clash\ for\ Windows-0.20.39-x64-linux /opt/clash
   ```

2. **启动软件**：
   ```bash
   cd /opt/clash
   ./cfw
   ```

### 4.2 配置桌面快捷方式

1. **创建 Desktop 文件**：
   ```bash
   cd ~/.local/share/applications
   nano clash_gui.desktop
   ```

2. **添加文件内容**：
   ```ini
   [Desktop Entry]
   Name=clash for linux
   Icon=/opt/clash/clash.png
   Exec=/opt/clash/cfw
   Type=Application
   ```

3. **赋予执行权限**：
   ```bash
   sudo chmod a+x clash_gui.desktop
   ```

### 4.3 配置系统代理

进入 **设置 (Settings) -> 网络 (Network) -> 网络代理 (Network Proxy)**，选择 **手动 (Manual)**，填写以下信息：

| 协议 | 主机 | 端口 |
| :--- | :--- | :--- |
| **HTTP 代理** | 127.0.0.1 | 7890 |
| **HTTPS 代理** | 127.0.0.1 | 7890 |
| **Socks 主机** | 127.0.0.1 | 7891 |

> **注意**：若需关闭代理，将“手动”改回“关闭”即可。
